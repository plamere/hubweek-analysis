HUBWeek Hackathon Dataset

For the HUBWeek hackathon, Spotify and Indaba have worked together to provide a detailed set of analysis data for a number of loops and stems in the Converse Rubber Tracks Sample library. This analysis provides all sorts of information about thousands of clips in the sample libarary.

Download the data

The dataset can be downloaded from indaba_analysis.json

Interpreting the analysis

The trid is the Echo Nest Track ID. Use this to retrive the detailed analysis of the track (see below). The file field contains the Indaba ID of the clip. Use this ID to associate the analysis data with a specific Indaba clip.

The track analysis includes summary information about a track including tempo, key signature, time signature mode, danceability, loudness, liveness, speechinesss, acousticness and energy. Note that for short samples, not all data can be calculated. Missing data will be returned with a null value.

Retrieving the detailed analysis

Associated with each clip is the detailed information about the song structure (sections) beat structure (bars, beats tatums) and detailed info about timbre, pitch and loudness envelope (segment). To retrieve this information use the track/profile call of the Echo Nest API to retrieve track information. Included in the information is an analysis url that can be used to retrieve the detailed analysis. The analysis URL expires after 10 minutes.

For details on how to interpret the data see the Analyzer Documentation.
